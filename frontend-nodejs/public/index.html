Production Policy and Enforcement Engine

Production checks that block risky releases before they ship.

Baseline helps engineering, platform, and security teams convert release expectations into deterministic controls. The product is CLI-first today, with real policy gates, readable reports, explain flows, AI-assisted remediation drafts, API-backed auth, and persisted audit history for teams that need one operational surface instead of scattered scripts and tribal process.

Install the CLI with npm i -g baselineprod-cli, then point it at the hosted API when you are ready to connect scans, sessions, and approvals.

Baseline ships frequent fixes right now, so keep your CLI current with npm i -g baselineprod-cli@latest before troubleshooting or trying new dashboard flows.

Current Product Surface

What Baseline already provides today.

Production-readiness policy pack

Readable reporting

Interactive Walkthrough

Explore the three product conversations already supported.

Detected: release-blocking policy violation

Baseline runs locally or in CI and returns a non-zero exit code when a repository misses a required production-readiness control.

  • Triggered checks include A1 through R1
  • Examples: secrets, tests, branch protection, deployment config, rollback docs
  • Output modes: human-readable gate, scan summary, SARIF, JSON

Generated: scaffolded remediation draft

AI support is limited to scaffolding and patch drafting. Policy outcomes remain deterministic and are not decided by the model.

  • baseline generate drafts files for common violations
  • baseline pr can generate, branch, commit, push, and attempt PR creation
  • Human review is still required before merge

Available: API-backed auth and operations

The current backend supports OIDC login, API keys, audit events, and dashboard summary/capabilities/activity endpoints for operational usage.

  • Human auth: Supabase/Auth0 via OIDC callback flow
  • Machine auth: bearer API keys
  • Persistence: SQLite for keys, events, and integration jobs

FAQ

Common questions before rollout.

Is Baseline real today, or mostly roadmap?

It is real today as a CLI-first product. The API, OIDC auth, API key flows, audit events, and dashboard backend endpoints are also implemented, while the dashboard UI is still being expanded.

Can it block CI/CD pipelines automatically?

Yes. baseline check and baseline enforce exit non-zero on blocking violations, which makes them suitable for gate steps in CI.

What does AI do, and what does it not do?

AI helps scaffold fixes and PR content. It does not decide whether a repository passes policy. Enforcement remains deterministic.

Do I need a database to adopt Baseline?

No for CLI-only usage. SQLite becomes useful when you want persistent API keys, audit events, and integration job queues on the API side.
