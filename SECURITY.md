---

## Response Timeline

We aim to:
- Acknowledge receipt within **48 hours**
- Provide a status update within **5 business days**
- Release a fix or mitigation as soon as possible

---

## Responsible Disclosure

We request that you:
- Give us reasonable time to fix the issue
- Avoid accessing or modifying user data
- Avoid service disruption during testing

---

## Security Best Practices (Project Side)

Baseline practices include:
- Environment variables for secrets (no secrets committed)
- HTTPS enforcement in production (when live)
- Input validation and output sanitization
- Dependency monitoring and updates
- Access control and authentication checks

---

## Third-Party Dependencies

Security of dependencies is monitored. Vulnerabilities are patched promptly.
